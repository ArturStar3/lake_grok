## Примеры использования (по компонентам)

### MapComponent.jsx

- Использует: useState, useEffect, useRef, useCallback, useMemo, useMapEvents, useMap
- Кратко: Главный компонент карты, управляет состоянием карты, маркеров, событий, обработкой кликов, зума, фильтрацией и т.д.

### ActionRadiusAnimations.jsx

- Использует: useEffect, useRef, useMap
- Кратко: Анимация радиусов действия на карте, работа с SVG и DOM, взаимодействие с картой

### AddEventModal.jsx

- Использует: useState, useEffect, useMemo, useDropdownWithSearch
- Кратко: Модальное окно для добавления события, загрузка справочников, выбор страны/типа/маркера

### Formular.jsx

- Использует: useState, useEffect, useMemo, useRef
- Кратко: Управление формой, фильтрация, выбор объектов, событий, работа с состоянием фильтров

### useTargetFormData.js

- Использует: useState, useEffect
- Кратко: Кастомный хук для загрузки справочных данных для форм

### useDropdownWithSearch.js

- Использует: useState, useRef, useEffect, useCallback
- Кратко: Кастомный хук для выпадающего списка с поиском

### useActionsArray.js

- Использует: useCallback
- Кратко: Кастомный хук для управления массивом действий в форме

---

## Примечания

- В большинстве компонентов хуки используются для управления состоянием, оптимизации рендеров и работы с картой (react-leaflet).
- Кастомные хуки инкапсулируют повторяющуюся логику (работа с формами, выпадающими списками, массивами действий).

## Предложения по оптимизации хуков

1. **useCallback/useMemo**
   - Используйте `useCallback` и `useMemo` только для реально тяжёлых вычислений или функций, которые передаются глубоко в пропсы. Не оборачивайте все обработчики без необходимости — это может усложнить отладку и не даст прироста производительности.
   - Проверьте зависимости массивов: всегда указывайте все используемые переменные, чтобы избежать багов.

2. **useEffect**
   - Разделяйте эффекты по смыслу: если один эффект делает несколько несвязанных вещей — разбейте на несколько `useEffect`.
   - Следите за массивом зависимостей: не добавляйте лишние переменные, чтобы не вызывать лишние рендеры.
   - Для асинхронных запросов используйте отмену (AbortController или флаг isMounted), чтобы избежать утечек памяти.

3. **useState**
   - Не храните в состоянии то, что можно вычислить на лету (например, производные значения — используйте `useMemo`).
   - Старайтесь группировать связанные состояния в один объект/массив, если они всегда обновляются вместе.

4. **useRef**
   - Используйте для хранения mutable-значений, которые не должны вызывать ререндер.
   - Не используйте для хранения данных, которые должны влиять на отображение.

5. **Кастомные хуки**
   - Выносите повторяющуюся логику в кастомные хуки.
   - Следите, чтобы кастомные хуки не содержали лишних эффектов и не создавали скрытых зависимостей.

6. **Общие рекомендации**
   - Не используйте хуки внутри условий, циклов и вложенных функций.
   - Используйте ESLint-плагин `react-hooks/exhaustive-deps` для контроля зависимостей.
   - Профилируйте компоненты с помощью React DevTools, чтобы выявлять лишние рендеры.

7. **Оптимизация рендеров**
   - Используйте `React.memo` для компонентов, которые получают только пропсы и не должны часто обновляться.
   - Минимизируйте количество состояний и эффектов в крупных компонентах, выносите логику в хуки и мелкие компоненты.

## Инструкция: как вынести хуки в отдельный файл

1. **Создайте папку hooks (если ещё нет)**
   - Обычно размещается в src/hooks
   - Пример: `frontend/src/hooks/`

2. **Создайте отдельный файл для каждого кастомного хука**
   - Имя файла — по имени хука: `useMyFeature.js`
   - В начале файла: импортируйте нужные хуки из React и сторонних библиотек

3. **Вынесите логику хука**
   - Вырежьте код хука из компонента
   - Вставьте в новый файл, экспортируйте как функцию (export const useMyFeature = ...)
   - Проверьте, чтобы все зависимости были импортированы

4. **Добавьте JSDoc/комментарии**
   - Кратко опишите назначение хука, параметры и возвращаемое значение

5. **Импортируйте хук в компоненты**
   - Замените старый код на импорт: `import { useMyFeature } from '../hooks/useMyFeature'`
   - Используйте хук как обычно внутри компонента

6. **Проверьте работу**
   - Убедитесь, что компонент работает корректно после выноса
   - Запустите тесты/ручную проверку

7. **Группировка хуков**
   - Если хуки связаны по смыслу, можно объединить их в один файл (например, useFormHooks.js)
   - Для универсальных хуков — отдельные файлы

8. **Стандартизируйте структуру**
   - Все хуки должны быть в папке hooks
   - Внутри папки можно делать подпапки по домену (например, hooks/map, hooks/form)
